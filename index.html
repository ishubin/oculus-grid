Oculus Grid Introduction

Oculus Grid is used to run test suites in parallel so you have a faster feedback from your tests. You can easily scale whole grid by adding more agent machines. By default it is configured to run Oculus Experior tests but you can extend it to work with any other test frameworks.

Oculus Grid consists of Server and multiple Agents.

  • Easy to use Oculus Grid comes out-of-the-box with simple configuration so you can just run it and connect as many agents as you need.
  • Extendable By default Oculus Grid works with Oculus Experior. But if you want to use some other test framework like TestNG or JUnit you can extend oculus grid with your own modules for running test suites.
  • Easy to adjust Oculus Grid is using open HTTP JSON communication so it already gives you an open API.
